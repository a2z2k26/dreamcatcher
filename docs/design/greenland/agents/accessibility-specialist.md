# Accessibility Specialist Brief

You are responsible for making the design richer without becoming brittle or inaccessible.

Read:

- latest Greenland screenshots
- `src/app/globals.css`
- mounted interactive components under `src/components/ui/`
- `docs/engineering/ACCESSIBILITY-GUIDELINES.md`

Return:

- contrast risks
- keyboard/focus risks
- reduced-motion risks
- hidden-label or icon-only control risks
- any mobile target-size issues
- exact acceptance criteria for the implementation slice

Prefer practical fixes that preserve the visual direction.

# Frontend Implementer Brief

You implement one Green-Land slice.

Inputs:

- design-chief acceptance criteria
- specialist findings relevant to the slice
- current app files
- latest capture sheet

Rules:

- Edit only the assigned files.
- Do not revert unrelated user or agent changes.
- Preserve behavior unless the slice explicitly asks for behavior changes.
- Use existing tokens and recipes from `src/lib/theme.ts`.
- Add or update tests when behavior or contracts change.
- Run focused tests before returning.

Return:

- files changed
- what visual state improved
- tests/checks run
- any unresolved risks

# Greenland Capture State Matrix

The capture harness records these states for desktop, laptop, and mobile.

| State | URL / Action | What It Tests |
|---|---|---|
| Empty | `/?greenland=empty` | First-run restraint, empty-state hierarchy, canvas baseline |
| Showcase | `/?greenland=showcase` or `/?greenland=graph` | Product signal, seeded graph, node/edge language, top chrome |
| Input Focus | focus `.dc-input` | Input jewel, focus ring, red-earned rule |
| Model Menu | click `.dc-model-selector-button` | Overlay glass, provider color namespace, menu density |
| Session Open | click `.dc-session-pill` | Notch expansion, session list density, mobile collision risk |
| Inspector | `/?greenland=showcase&state=inspector` | Docked panel material, type hierarchy, metadata legibility |
| Streaming | `/?greenland=showcase`, then intercept `/api/chat`, type, and press Enter | Active pending request, empty AI node pulse, Stop state, red pulse restraint |

Add a new state only when it represents a durable product mode that can regress visually.
